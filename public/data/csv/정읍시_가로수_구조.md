# 전북특별자치도 정읍시 가로수 노선별 현황 CSV 구조

- **파일**: `전북특별자치도 정읍시_가로수 노선별 현황_20240101.csv`
- **위치**: `dist/data/csv/` (또는 `public/data/csv/`)
- **인코딩**: UTF-8 (원본 EUC-KR인 경우 `npm run csv:utf8` 또는 `node scripts/csv-to-utf8.mjs "경로/파일명.csv"`로 변환)

## 컬럼 구조

| 컬럼명 | 설명 | 예시 |
|--------|------|------|
| 노선 | 도로(노선)명 | 내장로, 충정로, 천변로 |
| 시점 | 구간 시점 | 대림아파트, 구IC, 연지교 |
| 종점 | 구간 종점 | 부전 교차로, 대림아파트, 국유림관리소 |
| 조성년도 | 가로수 조성 연도 | 1985, 1996, 2004 |
| 연장 | 연장(km) | 11.5, 4.2, 3.5 |
| 수종 | 수종(복수 시 `+` 또는 `,` 구분) | 단풍나무+왕벚나무, 왕벚나무, "왕벚나무, 단풍" |
| 본수 | 가로수 본수(그루) | 2331, 679, 485 |
| 데이터기준일자 | 데이터 기준일 | 2024-01-01 |

## 전주시 CSV와의 차이

- **전주시**: 행당 한 노선, 컬럼이 `구`, `노선명`, `시점종점`, `가로수 그루수`, 그리고 **수종별 그루수 컬럼 다수** (은행나무, 느티, 왕벚나무 등).
- **정읍시**: 행당 한 노선, `노선`·`시점`·`종점`·`연장`·`수종`(문자열)·`본수` 등으로 **노선 단위 요약** 구조. 수종은 한 셀에 복합 문자열(예: `단풍나무+왕벚나무`).

## 행 수

- 헤더 1행 + 데이터 13행 = 총 14행 (공백 행 제외 시 15줄 기준 14행).

## 인코딩 변환

- 한글이 깨져 보이면 원본이 EUC-KR일 수 있음.
- 변환: `node scripts/csv-to-utf8.mjs "dist/data/csv/전북특별자치도 정읍시_가로수 노선별 현황_20240101.csv"`
- `csv-to-utf8.mjs`는 헤더에 `노선`·`본수`(또는 `시점`)가 있으면 정읍 형식으로 인식함.
